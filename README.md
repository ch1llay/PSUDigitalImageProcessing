# PSUDigitalImageProcessing

Лабораторные работы по курсу **«Цифровая обработка изображений»**, 3 семестр магистратуры ПГУ (преп. Максим Александрович).

Все работы выполнены на Python 3.13 с NumPy / OpenCV / scikit-image. Каждая папка — самостоятельный мини-проект: скрипт `main.py`, исходная картинка и сохранённый результат. Картинки для лаб 3–6 преподаватель прислал индивидуально.

| № | Папка | Тема | Что изучено |
|---|---|---|---|
| 1 | [lab1](lab1) | Основы NumPy | генерация случайной матрицы, обратная матрица (`linalg.inv`), поиск индексов максимумов (`argmax` / `unravel_index`) |

## Что в каждой лабе

### [lab1 — Основы NumPy](lab1)
Задание: сгенерировать случайную матрицу, инвертировать её и найти индексы максимальных элементов в обратной матрице.
- [main.py](lab1/main.py):
  - `generate_random_matrix` — случайная матрица 5×5 с проверкой на невырожденность (повтор генерации, если `det ≈ 0`);
  - обратная матрица через `np.linalg.inv` + проверка `A @ A⁻¹ ≈ E`;
  - `max_indices` — глобальный максимум (`argmax` + `unravel_index`), максимумы по строкам и столбцам.
- Результат — вывод в консоль, зафиксирован `seed=42` для воспроизводимости.

### [lab2 — Эквализация гистограммы](lab2)
Задание (одинаковое для всех): реализовать эквализацию гистограммы вручную.
- [main.py](lab2/main.py) — функция `equalize`: гистограмма яркостей → накопленная сумма (CDF) → нормировка → LUT `старая яркость → новая`. Панель 2×2: изображение и гистограмма до/после.
- [lenna.png](lab2/lenna.png) — исходное изображение.
- [result.png](lab2/result.png) — сравнение до/после: после эквализации гистограмма растянута на весь диапазон, контраст выше.

### [lab3 — Шум и фильтрация](lab3)
Задание: добавить шум, подавить его разными фильтрами и оценить восстановление метриками PSNR/SSIM.
- [main.py](lab3/main.py):
  - `add_salt_pepper`, `add_gaussian` — модели импульсного и гауссова шума;
  - фильтры: медианный, среднее, гауссов, билатеральный;
  - `metrics` — PSNR и SSIM (`skimage.metrics`) для каждого варианта.
- [b2a1f8f09f5d085eiokk.jpg](lab3/b2a1f8f09f5d085eiokk.jpg) — исходное изображение.
- [result_salt_pepper.png](lab3/result_salt_pepper.png) — на импульсном шуме лучший результат у **медианного** фильтра (SSIM ≈ 0.99).
- [result_gaussian.png](lab3/result_gaussian.png) — на гауссовом шуме лучше работают **линейные** фильтры.
- Вывод: выбор фильтра зависит от модели шума.

### [lab4 — Морфология и бинаризация](lab4)
Задание: убрать горизонтальные линии, перечёркивающие текст, и бинаризовать результат.
- [main.py](lab4/main.py) — функция `remove_lines`: инверсия → морфологическое **открытие** широким горизонтальным ядром (выделяет длинные линии, штрихи иероглифов отсеиваются) → `cv2.inpaint` (Telea) закрашивает линии → бинаризация Otsu и адаптивная.
- [as.png](lab4/as.png) — исходное изображение (текст с линиями).
- [output.png](lab4/output.png) — текст с удалёнными линиями; [output_binary.png](lab4/output_binary.png) — бинаризация Otsu.
- [result.png](lab4/result.png) — панель из 6 этапов обработки.

### [lab5 — Цветовая сегментация](lab5)
Задание: сегментировать объект, подбирая цветовые пороги вручную. Выделяем апельсины (синяя бутылка попасть не должна).
- [main.py](lab5/main.py) — перевод `BGR → HSV`, `cv2.inRange` с порогами `LOWER`/`UPPER` по тону/насыщенности/яркости, морфологические открытие и закрытие, наложение маски (фон чёрный).
- [18d33c106910765_5f9aa887279b1.jpg](lab5/18d33c106910765_5f9aa887279b1.jpg) — исходное изображение.
- [result.png](lab5/result.png) — панель: исходное / маска / сегментированные апельсины.
- Иллюстрирует компромисс пороговой сегментации: глубокие тени теряются, бежевая пробка изредка попадает в маску.

### [lab6 — Сегментация изображений](lab6)
Задание: сегментировать объекты разными методами и сравнить (картинка та же, что в lab5).
- [main.py](lab6/main.py):
  - `segment_watershed` — карта расстояний (`distance_transform_edt`) → локальные максимумы (`peak_local_max`) как маркеры → `watershed` разделяет слипшиеся апельсины, подсчёт и контуры объектов;
  - `segment_kmeans` — кластеризация пикселей по цвету (`sklearn.KMeans`), выбор «оранжевого» кластера;
  - region growing из центров найденных апельсинов.
- [segmentation_utils.py](lab6/segmentation_utils.py) — функция `region_growing` (наращивание области по похожести цвета, 4-связность).
- [result.png](lab6/result.png) — панель 2×3: исходное, цветовая маска, метки watershed, контуры, k-средних, region growing.
- Вывод: watershed надёжнее простой маски разделяет соприкасающиеся объекты; k-средних работает без ручных порогов, но чувствителен к тёплому цвету стола.

## Запуск

```powershell
# из корня проекта
pip install -r requirements.txt
```

Каждую лабу запускать из её папки (скрипт читает картинку рядом с собой):

```powershell
cd lab2
python main.py
```

Скрипты показывают результат через `matplotlib` и сохраняют его в `result.png` (или `output.png`) в папке лабы. Для прогона без открытия окон: `$env:MPLBACKEND='Agg'; python main.py`.
